## 需求背景

> 用2-3句话说明本次迭代的原因，最好附上数据或用户反馈支撑。

**本次背景：**
当前云存储套餐仅支持单次购买，用户每月需手动续费，操作繁琐且续费率偏低。为提升用户复购率和整体收入，需要增加连续订阅功能，通过更优惠的价格引导用户使用自动续费套餐，降低用户操作成本的同时提升业务收益。

### 需求1：连续订阅套餐展示优化

**需求来源：** 业务需求

**改动前（现状）：**
页面直接平铺展示4个套餐卡片：
- ¥9.9  7天事件存储（包月）
- ¥99.9 7天事件存储（包年）
- ¥19.9 30天事件存储（包月）
- ¥199.9 30天事件存储（包年）

用户每次需手动购买，无连续订阅选项。

**改动后（目标）：**
在套餐卡片区域顶部增加标签切换：
- 标签1：「单月购买」- 展示原价套餐
- 标签2：「连续订阅·更优惠」- 展示连续订阅优惠价

默认选中「连续订阅」标签，引导用户使用优惠套餐。每个套餐卡片展示：
- 优惠价（如 ¥8.8）
- 原价对比（如 原价 ¥9.9/月）
- 节省金额标签（如 省¥1.1/月）
- 「推荐」角标（首选项）

**交互说明：**

- 正常流程：
  1. 用户进入云存储购买页面
  2. 页面默认展示「连续订阅」标签内容，默认选中7天连续包月套餐
  3. 用户可切换「单月购买」标签查看原价
  4. 切换标签时，保持同存储时长套餐的选中状态（如7天包月 ↔ 7天连续包月）
  5. 用户点击选择目标套餐

- 边界条件：
  - 当用户已开通普通套餐时，单月标签下对应套餐置灰不可选
  - 当用户已开通连续订阅时，连续订阅标签下显示"自动续费中"标识
  - 当网络异常加载失败时，展示"加载失败，点击重试"

### 需求2：自动续费协议确认

**需求来源：** 业务需求 / 合规要求

**改动前（现状）：**
选择套餐后直接点击「去支付」按钮调起支付，无需额外确认。

**改动后（目标）：**
- 单月购买：保持原有逻辑，按钮文案为「去支付」
- 连续订阅：支付按钮上方增加协议勾选区域，按钮文案为「立即开通」

协议区域包含：
- 复选框（默认未勾选）
- 文案：已阅读并同意《自动续费服务协议》
- 协议链接（点击弹出协议详情弹窗）

**交互说明：**

- 正常流程：
  1. 用户选择连续订阅套餐
  2. 支付区域显示协议勾选框和「立即开通」按钮（置灰状态）
  3. 用户勾选协议，按钮变为可点击状态
  4. 用户点击「立即开通」调起支付流程

- 边界条件：
  - 未勾选协议时点击按钮，toast提示「请同意自动续费服务协议」
  - 点击协议链接弹出协议详情弹窗，内容可滚动阅读
  - 切换到单月购买标签时，隐藏协议区域，按钮变为「去支付」

**原型/示意图：**

```
┌────────────────────────────────────────────────┐
│                                                │
│  [x] 已阅读并同意《自动续费服务协议》          │
│                                                │
│  ¥8.8                    [立即开通]            │
│  支付宝                                        │
│                                                │
└────────────────────────────────────────────────┘
```

### 需求3：订阅管理入口（我的服务）

**需求来源：** 业务需求

**改动前（现状）：**
「我的服务」页面仅展示已购套餐基本信息，无订阅管理功能。

**改动后（目标）：**
在「我的服务」页面内展示：
- 当前的套餐类型
- 状态
  - 自动续费：自动续费中/已取消/扣款失败；⚠️：如果用户既没有主动取消自动续费，又没有扣款成功则显示“扣款失败”。如果支付宝返回已经取消（支付宝也可以取消），则该页面显示为“已取消”状态，同时下方按钮显示“重新开启续费”这样的状态
  - 单次购买：生效中/已失效。
- 下次扣款时间（如果是单次购买则显示“服务到期时间”）
- 扣款金额（如果是单次购买则显示“金额”）
- 扣款记录（如果是单次购买则不显示该模块）
- 取消续费入口
- 重新开启续费入口

⚠️：免费领取为时间比较短的单次购买套餐。

**交互说明：**

- 正常流程：
  1. 用户进入「我的服务」页面
  2. 已经开通服务的用户可以查看当前已经购买的服务内容
  3. 连续订阅用户显示下方取消自动续费与重新自动续费切换按钮，单次购买用户不显示该按钮
  4. 点击「取消自动续费」进行二次确认
  5. 取消成功后当前周期权益继续有效，到期后不再扣款
  6. 取消后用户可点击「重新开启续费」进行二次确认，二次确认弹窗内包含“已阅读并同意《自动续费服务协议》”，需同意后「确认开启」按钮才可点击，否则置灰显示。

- 边界条件：
  - 在该设备上从未购买过服务的用户不展示此模块
  - 扣款失败时发送Push通知提醒用户（标题：设备名称，提示语：“您的云存储服务“7天事件存储（连续包月）”自动扣款失败”）
