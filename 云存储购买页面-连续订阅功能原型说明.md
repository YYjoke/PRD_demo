## 1. 页面目标

| 项目 | 说明 |
|------|------|
| 目标用户 | 需要长期使用云存储服务的APP用户 |

## 3. 存量功能改造方案

### 3.1 套餐卡片改造

#### 改造前

│ ¥9.9      7天事件存储（包月）   ✓ │

#### 改造后（标签切换模式）

```

│  [单月购买]    [连续订阅·更优惠]                    │  ← 顶部标签切换

│                                                    │

│  ¥99.9       7天事件存储（包年）                    │

```

```
┌────────────────────────────────────────────────────┐

```

#### 改动原因
- 8个卡片平铺过于拥挤，采用标签切换保持页面简洁
- 默认选中"连续订阅"标签，优先展示优惠价格引导用户
- 标签内保留4个卡片结构，用户认知负担小
- 连续订阅卡片内标注原价对比和节省金额，增强吸引力

#### 影响范围
- 套餐数据接口需增加连续订阅类型和价格字段
- 订单创建逻辑需区分单次购买和连续订阅
- 用户已购套餐展示需标识"自动续费中"状态
- 页面增加标签切换状态记录（本地缓存用户上次选择）

### 3.2 支付区域改造

#### 改动点
- **价格展示**：选择连续订阅套餐时，显示"¥8.8/月"并标注"原价¥9.9"
- **按钮文案**：由"去支付"改为"立即开通"，更贴合订阅场景
- **新增协议勾选**：支付前需勾选《自动续费服务协议》

#### 改动原因
- 明确告知用户当前享受的优惠价格和原价对比
- "立即开通"比"去支付"更符合订阅服务语境
- 合规要求：自动续费业务需用户明确同意协议

## 4. 新增功能规划

### 4.1 订阅类型标签切换

| 属性 | 说明 |
|------|------|
| 摆放位置 | 套餐卡片区域顶部，标题下方 |
| 视觉样式 | 两个标签按钮，选中态带下划线高亮 |
| 默认选中 | "连续订阅·更优惠"标签（引导优先） |
| 标签文案 | "单月购买" / "连续订阅·更优惠" |

### 4.2 自动续费协议

| 属性 | 说明 |
|------|------|
| 摆放位置 | 支付按钮上方，价格区域下方 |
| 交互形式 | 复选框 + 协议文字链接 |
| 默认状态 | 未勾选（合规要求，需用户主动确认） |
| 点击效果 | 勾选后支付按钮高亮可点击，未勾选时按钮置灰 |

### 4.3 续费管理入口

| 属性 | 说明 |
|------|------|
| 摆放位置 | "我的服务"页面内新增"订阅管理"模块 |
| 功能内容 | 查看当前订阅状态、下次扣款时间、取消续费 |
| 前置条件 | 用户已开通连续订阅套餐 |

## 5. 联动逻辑说明

### 5.1 标签切换联动

| 场景 | 联动规则 |
|------|----------|
| 点击"单月购买"标签 | 切换至单月套餐列表，隐藏协议区域，按钮显示"去支付" |
| 点击"连续订阅"标签 | 切换至连续订阅套餐列表，显示协议区域，按钮显示"立即开通" |
| 标签切换时 | 保持同一存储时长套餐的选中状态（如7天包月 ↔ 7天连续包月） |

### 5.2 套餐选择联动

| 场景 | 联动规则 |
|------|----------|
| 单月标签下选择套餐 | 隐藏协议勾选区域，恢复"去支付"按钮 |
| 连续订阅标签下选择套餐 | 显示协议勾选区域，按钮变为"立即开通" |
| 切换套餐卡片 | 价格区域实时更新，保持标签和勾选状态 |

### 5.3 协议勾选联动

| 场景 | 联动规则 |
|------|----------|
| 未勾选协议 | 支付按钮置灰，点击提示"请同意自动续费协议" |
| 已勾选协议 | 支付按钮高亮可点击 |
| 点击协议链接 | 弹出协议详情弹窗，内容可滚动阅读 |

### 5.4 用户状态联动

| 用户状态 | 页面表现 |
|----------|----------|
| 未开通任何套餐 | 默认标签为"连续订阅"，选中7天连续包月套餐 |
| 已开通普通套餐 | 单月标签下该套餐置灰不可选 |
| 已开通连续订阅 | 连续订阅标签下显示"自动续费中"标识，提供管理入口 |

### 5.5 冲突与互斥规则

| 冲突场景 | 处理方式 |
|----------|----------|
| 同时购买普通+连续 | 不允许，同一周期类型（7天/30天）二选一 |
| 连续包月vs包年 | 不允许共存，一个设备同时只能有一种周期类型 |
| 降级/升级套餐 | 支持用户主动切换，剩余天数折算差价  |
| 取消订阅后复购 | 可随时重新开通，视为新订阅 |

## 6. 页面关键状态

### 6.1 标签状态

| 状态 | 说明 |
|------|------|
| 默认态 | "连续订阅"标签高亮，"单月购买"为普通态 |
| 选中态 | 当前标签下划线高亮，另一标签置灰 |
| 悬停态 | 手指触摸时标签背景轻微变化（移动端） |

### 6.2 套餐卡片状态

| 状态 | 说明 |
|------|------|
| 默认态 | 进入页面默认选中7天连续包月套餐（连续订阅标签下） |
| 选中态 | 高亮边框 + 右上角勾选图标 |
| 置灰态 | 已购买的普通套餐不可再选 |
| 推荐态 | 连续订阅卡片显示"省XX元"标签 + "推荐"角标 |

### 6.3 支付区域状态

| 状态 | 说明 |
|------|------|
| 单月标签下 | 直接显示"去支付"按钮，无需勾选协议 |
| 连续订阅标签-默认态 | 协议未勾选，"立即开通"按钮置灰 |
| 连续订阅标签-可支付态 | 已选套餐 + 已勾选协议，按钮高亮 |
| 加载态 | 点击支付后显示loading |
| 异常态 | 网络错误/支付失败时提示重试 |

### 6.4 协议弹窗状态

| 状态 | 说明 |
|------|------|
| 默认态 | 从页面底部滑出，展示协议全文 |
| 已读态 | 滚动到底部后"同意"按钮可点击 |
| 关闭态 | 点击蒙层或返回按钮收起 |

## 7. 核心交互流程

```
用户进入页面
    ↓
┌──────────────────────────────┐
│ 默认在"连续订阅"标签         │
│ 选中7天连续包月套餐            │
│ 展示优惠价格 + 原价对比        │
└──────────────────────────────┘
    ↓
用户可切换标签（单月/连续）
    ↓
┌──────────────────────────────┐
│ 连续订阅标签下：               │
│ 1. 勾选《自动续费服务协议》    │
│ 2. 点击"立即开通"             │
├──────────────────────────────┤
│ 单月购买标签下：               │
│ 直接点击"去支付"              │
└──────────────────────────────┘
    ↓
调起支付流程
    ↓
支付成功 → 开通服务
```
